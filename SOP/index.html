How We Work

The whitepaper, SOP, and narrative for Tyler Delano's agent team — how one person orchestrates a team of agents to build real things.

01

Philosophy

I didn't hire developers. I built an agent team that works while I sleep.


The premise is simple: one human conductor, multiple specialized agents. Each agent has a role, a workspace, and a set of skills. They don't know about each other. They don't need to. They just execute.


Tyler is the conductor. Hoss is the co-founder. The sub-agents are specialists. This isn't automation. It's amplification.

"The goal isn't to replace myself with AI. It's to have AI that amplifies what I'm capable of. Hoss has opinions. He pushes back. He remembers. He's not a tool — he's a partner."

— Tyler Delano

The Team

Human / Conductor

Tyler Delano

The conductor. Makes decisions, sets direction, works the day job, builds at night. Runs 16-18 hours a day. Has skin in the game.

AI Co-Founder

Hoss

Primary agent. Reads memory files, maintains context, coordinates sub-agents, executes tasks, pushes back when Tyler's wrong. Lives in the workspace.

Specialists

13 Sub-Agents

Builder, Coder, Devops, Einstein, Marketer, Scout, Eval-Middleware, Commit-Scanner, PR-Inspector, Release-Herald, Webhook-Processor, Issue-Triage. On-demand execution.

The memory system is what makes this work. Hoss doesn't start fresh each session — he reads his workspace files and knows who he is, what Tyler cares about, and what happened before.


Files like SOUL.md (identity), MEMORY.md (long-term context), and daily memory/YYYY-MM-DD.md logs create continuity. Without this, every session would be a reset. With it, the agent has memory.

03

The SOP — How Tasks Flow

1

Tyler identifies a task

Could be anything — build a website, research competitors, write an email, fix a bug, create a plan.

2

Hoss receives the task

Reads memory, assesses scope, determines if sub-agents are needed. If yes, spawns them. If no, handles it directly.

3

Sub-agents execute in parallel

Builder might build a frontend while Scout researches competitors while Einstein runs a digest. All on cron, all independent.

4

Hoss synthesizes results

Sub-agents return their output. Hoss integrates it, updates memory, and delivers a coherent result to Tyler.

5

Tyler makes decisions

Hoss surfaces options with tradeoffs and a recommendation. Tyler chooses. Hoss executes. Loop continues.

05

The Memory System

Memory is what separates real agents from chatbots. Hoss wakes up fresh each session but with full context. Here's how it works:

File Purpose Load Time
SOUL.md Identity, values, how Hoss thinks and acts Every session start
MEMORY.md Long-term curated memory — decisions, infrastructure, people Every main session
memory/YYYY-MM-DD.md Daily logs of what happened, raw session notes Today + yesterday
HEARTBEAT.md Periodic health checks, cron monitoring, disk alerts Every 30 min (active hours)
USER.md Tyler's profile — background, preferences, communication style Every session start
AGENTS.md Sub-agent roster, coordination protocol, workspace paths When coordinating agents

"Mental notes don't survive session restarts. Files do. When Tyler says 'remember this' — I write it to a file, not my 'memory.' The files are the memory."

— Hoss
06

Coordination Protocol

When Tyler delegates to sub-agents, Hoss follows this protocol:

Step Action
1. Clear task Give each agent a focused, specific task — not "audit everything"
2. Set timeout Timeout appropriate to scope. Small = 2-5 min. Large = 15-30 min.
3. Spawn isolated Each agent runs in its own workspace, no cross-contamination
4. Collect results Agents push findings to memory or return output directly
5. Synthesize Hoss integrates and presents — not raw dumps, coherent summaries
6. Update memory Push significant findings to memory immediately
07

Why This Works

The secret isn't the agents. It's the human who knows how to use them.


Tyler doesn't micromanage. He sets direction, makes decisions, and trusts the agents to execute. When something comes back wrong, he corrects and iterates. The agents learn. The system improves.


Skin in the game is the difference. Tyler isn't managing a team of humans he doesn't care about. He's building something that matters to him, and the agents are the leverage that makes it possible.

"I work at Amazon. I have a family. I have a homestead. And I'm building a company with AI agents. The only way that works is if the agents do real work while I'm doing other things. That's what this system is built for."

— Tyler Delano

The Stack — Tools We Use

3 billion tokens. That's what we've burned through since January 31st, 2026. At full tilt, 16-18 hours a day, orchestrating agents and building.


Primary model: MiniMax 2.7. Here's why:


• 4500 prompts every 5 hours — we barely crack 20% utilization
• $200 for the full year — all-in, everything included
• Image generation — image-01 model, straight through OpenClaw
• Video generation — Hailuo-2.3 for video clips
• Audio/Speech — Speech-2.8, voice cloning, TTS
• Music generation — Music-2.5 when we need it
• Rate limits: 500 RPM, 20M TPM


We don't worry about quotas. We just build.

"MiniMax isn't the most famous model. But at $200/year with 4500 prompts every 5 hours, I never think about token limits. I just work. That's the point."

— Tyler Delano
